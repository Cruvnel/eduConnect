# Entregas 3 e 4 — Front-end Avançado e Controle de Versão (React e GitHub)

Nesta etapa final do projeto serão desenvolvidas **interfaces modernas utilizando React**, e a utilização do **controle de versão com Git e GitHub**.

O objetivo é tornar o sistema **mais escalável, reutilizável e colaborativo**, aplicando boas práticas de desenvolvimento frontend e fluxo de trabalho profissional.

Principais focos desta etapa:

- Componentização da interface
- Gerenciamento de estado
- Sistema de rotas e navegação
- Integração com a API desenvolvida no backend
- Upload de arquivos
- Estratégias de cache
- Versionamento do projeto com GitHub
- Documentação do sistema

---

# Componentização

O frontend será desenvolvido utilizando **React**, permitindo a criação de **componentes reutilizáveis** para manter o código organizado e escalável.

## Biblioteca de Componentes

Será criada uma pequena **biblioteca de componentes reutilizáveis**, utilizada em todo o sistema.

### Exemplos de componentes

- Botões
- Inputs
- Cards
- Modais
- Tabelas
- Formulários
- Barras de navegação
- Alertas e notificações

### Estrutura sugerida



Cada componente deverá ser **reutilizável e desacoplado**, facilitando manutenção futura.

---

# Gerenciamento de Estado

O gerenciamento de estado será utilizado para controlar dados compartilhados entre componentes.

### Possíveis abordagens
- **React Context API**
- **React Query (para dados da API)**

### Exemplos de dados controlados por estado

- Usuário autenticado
- Tema claro/escuro
- Dados de alunos e professores
- Notificações do sistema
- Dados do dashboard
